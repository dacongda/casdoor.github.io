Keycloak

JBoss KeyCloak システムは、SAML および OpenID Connect を介したアプリケーションとの統合をサポートする、広く使用されているオープンソースの ID 管理システムです。 また、LDAP などの他のプロバイダーや、SAML または OpenID Connect をサポートする他のプロバイダー間のアイデンティティブローカーとして動作することもできます。

以下は、KeyCloak で新しいクライアントエントリを設定し、Casdoor が KeyCloak 設定を介してアクセス権を付与された KeyCloak ユーザーによるUIログインを許可するように設定する方法の例です。

Configure Keycloak​

この例に特化した設定の選択と仮定があります。

  • Casdoorをローカルで開発モードとして実行していると仮定しましょう。 Casoor UI is available at: http://localhost:7001 and server is available at http://localhost:8000. 必要に応じて適切なURLに置き換えてください。
  • ローカルで Keycloak を実行していると仮定しましょう。 Keycloak UI is available at: http://localhost:8080/auth.
  • これに基づいて、このデプロイメントの SP ACS URL は以下のようになります: http://localhost:8000/api/acs.
  • SP エンティティIDは以下のURLを使用します: http://localhost:8000/api/acs.

デフォルトのレルムを使用するか、新しいレルムを作成します。

Keycloakにクライアントエントリを追加​

Keycloak ドキュメント にある Keycloak クライアントの詳細を参照してください。 :::

メニューの クライアント をクリックし、 Create をクリックして Add Client ページに移動します。 以下のように記入してください。

  • Client ID: http://localhost:8000/api/acs - 後でCasdoor設定で使用されるSPエンティティIDになります。
  • Client Protocol: saml.
  • Client SAML Endpoint: http://localhost:8000/api/acs. - このURLは、KeycloakサーバーにSAMLリクエストと応答を送信する場所です。 一般的に、アプリケーションにはSAMLリクエストを処理するための1つのURLがあります。 クライアントの format@@0 タブで複数の URL を設定できます。

保存 をクリックします。 このアクションはクライアントを作成し、 設定 タブに移動します。

設定の以下のリスト:

  1. Name - Casdoor. これはKeyCloak UIでKeycloakユーザーにフレンドリーな名前を表示するためにのみ使用されます。 好きな名前を使えます。
  2. 有効 - 選択。
  3. Authn ステートメント - 選択します。
  4. Sign Documents - Select on.
  5. サインアサーション - オフを選択します。
  6. Encrypt Assertions - Select off.
  7. Client Signature Required - Select off.
  8. 強制名 ID フォーマット - オンを選択します。
  9. Name ID Format - ユーザ名を選択します。
  10. 有効なリダイレクト URI - 追加 http://localhost:8000/api/acs.
  11. Master SAML Processing URL - http://localhost:8000/api/acs.
  12. 微粒子SAMLエンドポイント設定
    1. Assertion Consumer Service POST Binding URL - http://localhost:8000/api/acs.
    2. Assertion Consumer Service Redirect Binding URL - http://localhost:8000/api/acs.

設定を保存します

Installation タブをクリックします。

Keycloak <= 5.0.0 の場合は、 SAML メタデータ IDPSSODescriptor を選択し、メタデータをコピーします。

Keycloak 6.0.0以降では、 Format Option - Mod Auth Mellon files Download をクリックします。 ダウンロードした.zipを解凍し、 idp-metadata.xml を見つけてメタデータをコピーします。

Casdoorで設定​

Casdoorに新しいプロバイダを作成します。

SAMLとしてカテゴリを選択し、 Keycloak と入力します。 メタデータの内容をコピーし、 メタデータ 入力に貼り付けます。 The values of Endpoint, IdP and Issuer URL will be generated automatically after clicking the Parse button. 最後に、ボタンをクリックします 保存.

If you enable the Client Signature Required option in Keycloak and upload the certificate, please enable the Sign request option in Casdoor. :::

Casdoorで設定したいアプリケーションを編集します。 追加されたプロバイダを選択し、ボタン 保存をクリックします。

効果を検証する​

設定したアプリケーションに移動すると、ログインページにKeycloakアイコンが表示されます。

アイコンをクリックし、Keycloakのログインページにジャンプし、認証後にCasdoorに正常にログインします。
