概览

Cassdoor 可以使用其他 OAuth 应用程序作为登录方法。

现在，Casdoor 支持许多OAuth 应用程序提供者。 供应商的图标将在添加到Casdoor后显示在登录和注册页面中。 这里是供货商连锁店支持：

GoogleGitHubFacebook推特LinkedInWeiboWeChat腾讯QQ正在说话中Gitee电子邮件地址短信
✅✅✅🚧✅✅✅✅✅✅✅✅

我们将向您展示如何申请第三方服务并将其添加到连锁店。

申请成为开发者​

在此之前，你需要理解一些一般概念。

  • 重定向Url, 认证后重定向地址, 填写您的应用程序地址, 例如 https://forum.casbin.com/
  • 范围，用户授予您的权限，如基本个人资料，电子邮件地址和帖子及其他。
  • ClientId/AppId, ClientKey/AppSecret, 这是最重要的信息 而且这是您在申请开发者帐户后需要得到的， 无法与任何人分享 密钥/密钥。

Twitter(仍在工作🚧)​

Twitter的应用程序步骤有点麻烦，官方限制有点严格， 因此申请开发者帐户可能比其他第三方平台更困难。

访问 开发者门户，如果您没有帐户，请注册。 Twitter需要知道您正在申请哪些开发者帐户。 你必须仔细填写它，否则它将不会通过。

申请被批准后，创建申请书，填写回调地址和其他信息。 您需要做两件事，将被设置为 认证设置 部分。

  • 手动打开 3-腿OAuth，用Twitter登录，代表其他帐户发布Tweets等等。
  • 启用 向用户请求电子邮件地址以获取用户电子邮件地址。

Weibo✔️​

Weibo的开发者帐户应用程序并不困难，但速度相对较慢，需要大约2-3天。

访问 开发者网站，填写基本信息并等待长时间审查...

审核通过后，您可以获得客户端ID和客户端密钥。

WeChat:very_check_mark:​

访问 WeChat 开发者平台，注册为开发者， 在您的 web 应用程序或您的移动应用程序获得批准后，您就会得到App Id 和 App Secretary。

欲了解更多详情，请访问 WeChat 打开平台。

腾讯的QQ:重型检查标记：​

访问QQ认证平台 - 连接 QQ。

首先您需要在审核通过后申请 成为开发者。 遵循平台的指示，并获取您的客户端ID和客户端密钥。

DingTalk:very_check_mark:​

访问 DingTalk 开发者平台 并登录使用您的 DingTalk 帐户。 在输入平台后，按照平台的指示，您将会得到您的客户端ID和客户端的密钥。

欲了解更多详情，请访问 DingTalk 开发者 docs。

添加 OAuth 提供商​

  1. 导航到您的城堡索引页面
  2. 点击顶部栏中的 供应商
  3. 点击 添加，然后您可以在列表顶部看到一个新的提供商
  4. 点击新的提供商修改它
  5. 选择 OAuth 在 类别 中
  6. 选择 需要的 OAuth 提供商
  7. 填写最大的导入信息， 客户端 ID 和 客户端密钥

应用中​

  1. 点击 应用于顶部栏中的 并选择一个应用程序，编辑
  2. 点击提供商添加按钮，选择您刚刚添加的提供商
  3. 修改提供商的权限，例如允许注册、登录和取消绑定
  4. 完成！
